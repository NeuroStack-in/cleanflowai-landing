CleanFlowAI — Revised UX Proposal

B2B enterprise data quality platform used by business people (accountants, data stewards, ops teams). The design must be professional, minimalist, and user-friendly — not a dark hacker terminal, but a clean productivity tool they trust with their data.

1 — Theme Direction
Current: Dark "Command Center"
Looks cool, wrong audience
Dark UIs work for developer tools (VS Code, Terminal, Grafana). But 81% of business users prefer light mode for productivity apps. Dark themes increase eye strain during extended data review sessions. The "hacker aesthetic" signals "technical tool" — not "business platform."
Proposed: Clean Light Professional
Trust, clarity, productivity
Inspired by Linear, Notion, Atlan, and Snowflake's console. Soft neutral backgrounds (#F8F9FB, not pure white). High-contrast text for readability. Blue primary for trust/authority. Teal accent for data health indicators. Minimal borders, generous whitespace.
Research insight
"Muted tones of gray create a serene, businesslike environment that makes navigation more natural and shows important information more clearly." — Minimalist UI Design for SaaS (Uitop, 2025). Dark mode should remain available as an option, but light mode should be the default.

Component Style Guide
Theme
Buttons: Solid fill for primary actions (blue), outline for secondary, ghost for tertiary. Consistent 8px radius. No gradients, no glows.
Cards: White surface, 1px #E5E7EB border, 12px radius, no shadows (or very subtle shadow-sm).
Tables: No zebra stripes. Subtle bottom border per row. Hover state: #F3F4F8 background. Header: uppercase 11px, gray-500 text.
Badges/Pills: Light tinted backgrounds with matching text (e.g., blue-50 bg + blue-600 text). No heavy borders.
Spacing: 4px base grid. 16px component padding. 24px section gaps. Generous whitespace.

Summary of Changes
Theme: Light-first professional theme (Inter + IBM Plex Mono, blue primary, soft neutral backgrounds)
Sidebar: Light sidebar with attention badges and active job indicators
Dashboard: KPI trends, Action Required panel, live processing status
Files: Pipeline stepper replacing text badges, one-click Quick Process
Export: Inline dropdown replacing 3-dialog chain
Onboarding: Getting-started checklist for empty states

What we're NOT changing: All existing functionality, data flows, API calls, and business logic remain identical. These are UX/UI improvements only.
