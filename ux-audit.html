1 Dashboard Page

⚠ Problem High Impact
Dashboard is read-only — zero actionable items
Users see charts and KPIs but cannot act on anything. No "files need attention" with direct links. No "click to remediate". The attention banner links to /files but loses context. Users must mentally map numbers → navigation → action.
✓ Proposed Fix
Add "Action Required" panel with direct deep-links
Replace the generic attention banner with a prioritized action queue: "3 files have quarantined rows — click to open editor". Each item is a clickable card that opens the relevant dialog directly. Failed files link straight to retry.

⚠ Problem High Impact
No file-level progress visible from dashboard
If a file is currently processing (DQ_RUNNING), the dashboard shows nothing about it. Users have to navigate to Files page to check. The Activity Feed shows past events only, not live processing status.
✓ Proposed Fix
Add "Live Pipeline" status strip below KPIs
Show currently-processing files with a live progress indicator directly on the dashboard. When nothing is processing, this strip auto-hides. Clicking it navigates to the file's detail view.

⚠ Problem Medium
KPI cards don't show trend/change direction
"Avg DQ Score: 87.5%" — is that better or worse than last week? The cards show a snapshot with no context. Users can't tell if quality is improving.

2 Files / Data Catalog Page

⚠ Problem High Impact
File status is a static badge — no progression visualization
Files go through: UPLOADED → QUEUED → DQ_RUNNING → DQ_FIXED. But the table shows a single text badge. Users can't see where in the pipeline a file is, or how much work remains. "DQ_RUNNING" tells you nothing about progress.
✓ Proposed Fix
Replace status badge with a mini pipeline stepper
Show a 3-dot stepper (Upload → Process → Complete) with the current step highlighted. For DQ_RUNNING, show an animated pulse on the "Process" dot. Failed states show the step that failed in red.

⚠ Problem High Impact
13+ dialogs stacked in files-page-dialogs.tsx — modal maze
The files page renders 13 different modals (delete, rename, column selection, rules, confirm columns, details, download, ERP transform, push to ERP, wizard, column export, export, profiling, import, quarantine editor). Users can trigger dialog → dialog → dialog chains. This is a navigation nightmare.
✓ Proposed Fix
Consolidate into a slide-over detail panel + unified action drawer
Replace the "click row → open full-screen dialog" pattern with a persistent right panel (slide-over) showing file details. Actions (export, process, edit) become a single action drawer at the bottom of this panel. No more modal stacking.

⚠ Problem Medium
No drag-and-drop anywhere on the files page
Users must click "Import" → wait for dialog → then drag a file into the dialog's drop zone. The main page surface — which is often empty for new users — doesn't accept file drops. This is a missed opportunity for the most common action.
✓ Proposed Fix
Make the entire files page a drop zone
When a user drags a file over the page, show a full-page drop overlay ("Drop to upload and process"). On drop, auto-open the processing wizard. This is especially powerful when the page is empty (first-time experience).

3 Sidebar & Navigation

⚠ Problem Medium
Sidebar shows no live context — just static links
The sidebar is a plain nav menu. It doesn't show: how many files are processing, how many jobs are running, if anything needs attention. Users navigate blind — they don't know where to go until they get there.
✓ Proposed Fix
Add contextual count badges + status indicators
Show a red badge "3" on Data Catalog when 3 files need attention. Show a green pulse dot on Jobs when active jobs are running. These cost almost nothing to implement but drastically improve wayfinding.

4 Dialogs & Workflows

⚠ Problem High Impact
Processing wizard is 6 steps but most users only need 1
The wizard has: Source → Columns → Profiling → Settings → Rules → Process. But 80% of users just want "process with defaults". The wizard forces everyone through 6 screens even when the defaults are fine. Power users need the granularity, but it shouldn't be mandatory.
✓ Proposed Fix
Add "Quick Process" one-click + "Advanced" expandable wizard
After upload, show two paths: "Process Now" (one click, default settings) and "Configure Processing →" (opens the full wizard). The quick path skips all optional steps. Advanced users can always access the full wizard.

⚠ Problem Medium
Export flow requires 3+ clicks through nested dialogs
To export: click Export action → choose format in ExportDialog → (optional) column selection → download. That's 3 dialogs for what should be the simplest action on the platform.
✓ Proposed Fix
Inline export dropdown directly on the action button
Replace the export dialog chain with a dropdown menu on the export button: CSV | Excel | JSON. One click to download. "Advanced export..." at the bottom opens the column selection if needed.

5 Jobs Page

⚠ Problem Medium
Job run history is hidden behind an expand click
To see if a job's last runs succeeded, you must click the expand chevron on each row. There's no at-a-glance view of "which jobs are healthy vs. failing".
✓ Proposed Fix
Add inline sparkline showing last 7 runs
In the table, add a tiny sparkline column showing the last 7 run results as colored dots (green = success, red = fail, gray = not run). At a glance, users see patterns — "this job fails every Monday".

6 Empty States & Onboarding

⚠ Problem High Impact
First-time user sees an empty dashboard with zero guidance
New user lands on Dashboard: empty KPIs (all "0" or "—"), empty charts, empty activity feed. There's no "Welcome! Here's how to get started". The most critical moment (first session) is completely wasted.
✓ Proposed Fix
Replace empty dashboard with an onboarding checklist
When files.length === 0, show a getting-started card instead of empty charts:

☐ Upload your first file
☐ Run data quality processing
☐ Review and fix quarantined rows
☐ Export clean data

Each step links directly to the relevant page/action. The card disappears once all steps are completed.
⚠ Problem Medium
Empty tables show "No files yet" but don't explain why or what to do
Files table empty state: icon + "No files yet" + "Import a file to start analyzing data quality" + button. This is minimal but doesn't educate. What file formats? What happens after import? Users don't know what to expect.
✓ Proposed Fix
Rich empty state with format chips + mini workflow preview
Show supported formats as clickable chips (CSV, Excel, JSON). Below that, a 3-step mini diagram: Upload → AI processes → Clean data. This sets expectations and makes the empty state feel like a feature, not a void.
